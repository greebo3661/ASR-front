# ASR Studio Frontend

ASR Studio Frontend — это современный, быстрый и минималистичный Single Page Web-интерфейс (SPA) для взаимодействия с бэкенд-системами автоматического распознавания речи (ASR – Automatic Speech Recognition).

Проект использует React, TypeScript, Vite и профессиональный UI на базе Tailwind CSS для обеспечения максимального комфорта при загрузке аудио и получении результатов транскрибации.

![ASR Studio Preview](https://via.placeholder.com/800x450.png?text=ASR+Studio+UI) <!-- Замените на реальный скриншот -->

## ✨ Ключевые возможности

*   **Абсолютно новый Premium UI**: Интуитивный, "стеклянный" (Glassmorphism) дизайн с плавной анимацией взаимодействия.
*   **Drag & Drop загрузчик**: Удобное перетаскивание медиа-файлов (аудио/видео) прямо в браузер.
*   **Умное управление статусами**: Отображение нагрузки на "AI-движки" (сервисы распознавания) через систему скрытых бейджиков.
*   **Live-прогресс**: Поллинг статуса выполнения транскрибации (без race conditions благодаря `AbortController`).
*   **Продвинутый просмотр результатов**: 
    * Вкладка со **сплошным текстом**.
    * Вкладка с **сегментами (субтитрами)** и таймкодами (формат `[00:00 - 01:23]`).
    * Скоростное копирование в один клик.
    * Экспорт результатов в форматы `.txt` и `.srt` (субтитры).
*   **Без CORS проблем**: В комплекте настроенный `Nginx` для прозрачного проксирования API бэкенда.

## 🚀 Как запустить (Docker / Продакшен)

Рекомендуемый способ запуска интерфейса в инфраструктуре — через Docker Compose. Это мгновенно поднимает Nginx, раздает статику и проксирует запросы на бэкенд-нейросети.

### 1. Клонирование репозитория
```bash
git clone https://github.com/твое_имя/asr-frontend.git
cd asr-frontend
```

### 2. Настройка окружения
Скопируйте пример файла окружения и укажите адрес вашего сервера-распознавания речи.

```bash
cp .env.example .env
```
Откройте файл `.env` и измените переменную `ASR_API_BASE` на IP-адрес/домен вашего ASR REST API:
```dotenv
ASR_API_BASE="http://192.168.x.x:19000" # Или http://backend:19000 если бэкенд в той же docker-сети
```

### 3. Запуск контейнера
Сборка и старт образа за секунды:
```bash
docker compose up --build -d
```
Сайт станет доступен по адресу `http://localhost:19000` (порт настраивается в `docker-compose.yml`).

## 💻 Разработка (Локально)

Если вы хотите вносить изменения в дизайн или функционал локально на своей машине:

### Требования
* Node.js v20+

### Установка зависимостей
```bash
npm install
```

### Запуск Dev-сервера
Вам не нужен Docker для фронтенд-разработки. Просто укажите адрес внешнего API при запуске скрипта, чтобы Nginx не был нужен:

```bash
# Вариант 1: Сконфигурируйте переменную в .env-файле (VITE_API_BASE) и запустите
npm run dev
```

Откройте `http://localhost:5173` в вашем браузере.

## 📂 Структура проекта

* `src/` - Исходный код приложения
  * `api.ts` - Изолированный модуль сетевых HTTP-запросов (`fetch`).
  * `types.ts` - TypeScript интерфейсы.
  * `App.tsx` - Главная точка входа, хранит общие стейты, реализует логику Long-polling.
  * `components/` - Переиспользуемые визуальные кубики `UploadForm.tsx`, `JobProgress.tsx`, `JobResult.tsx`.
* `Dockerfile` - Инструкции для Docker-сборки.
* `nginx.conf` - Ключевой конфиг Nginx, который раздает UI и проксирует `/api` запросы с клиента на реальный бекенд, спасая приложение от CORS блокировок.
* `entrypoint.sh` - Bash-скрипт инъекции `.env` внутрь Nginx перед его запуском.
